Plan: Refactor, Test Infrastructure, and Hardening

Context

The extension works (M1-M3 complete) but is prototype-quality: significant code duplication across the screenshot
pipeline and fiber walk logic, zero real tests, inconsistent error handling, sync I/O in the hot path, and dead
code. Before marketplace release (or adding Tailwind/Next.js support), the codebase needs to be hardened. This
plan combines architectural refactoring, test infrastructure, and error handling into a single pass — they're
deeply intertwined and each step leaves the codebase compilable.

---

Step 1: Extract shared screenshot constants

Create src/screenshotConstants.ts with MAX_BYTES, QUALITY_STEPS, MAX_CAPTURE_WIDTH, MAX_CAPTURE_HEIGHT. Remove
duplicate declarations from src/renderer.ts (lines 30-33) and src/devServerRenderer.ts (lines 6-11). Both files
import from the new module.

Files: create src/screenshotConstants.ts, modify src/renderer.ts, modify src/devServerRenderer.ts

Step 2: Extract adaptive screenshot pipeline

Create src/screenshotPipeline.ts with a single captureAdaptiveJpeg(target, ctx) function that takes a Playwright
Locator | ElementHandle and BrowserContext, runs the quality-stepping loop, and applies the resize fallback if
needed. Returns a Buffer guaranteed to fit within the base64 limit.

- src/renderer.ts: remove resizeBuffer(), replace quality loops in renderElement() and compressImageFile() with
  captureAdaptiveJpeg() calls
- src/devServerRenderer.ts: remove inline quality loop + resize block (lines 288-318), replace with
  captureAdaptiveJpeg() call

Files: create src/screenshotPipeline.ts, modify src/renderer.ts, modify src/devServerRenderer.ts

Step 3: Extract scoring algorithm and JSX patch

Create two small modules:

- src/fiberScoring.ts — exports scoreFiber(candidateLine, targetLine) as a pure function. The page.evaluate
  callbacks in devServerRenderer.ts keep their inline copies (browser boundary constraint) but the canonical tested
  version lives here.
- src/jsxDevPatch.ts — exports the JSX_DEV_RUNTIME_PATCH string constant. Remove the duplicate from
  devServerRenderer.ts (lines 24-52), import from the new module.

Files: create src/fiberScoring.ts, create src/jsxDevPatch.ts, modify src/devServerRenderer.ts

Step 4: Set up vitest for unit tests

Add vitest as a dev dependency. Create vitest.config.ts that includes src/**/\*.unit.test.ts and excludes
src/test/** (the VS Code EDH integration tests). Add test:unit and test:unit:watch scripts to package.json.

Convention: unit tests use \*.unit.test.ts suffix, co-located next to the source file they test.

Files: create vitest.config.ts, modify package.json

Step 5: Write unit tests

Export findDeepestAtOffset and elementIdentityFromNode from htmlAnnotator.ts (they're pure internal functions, no
public API impact).

Create test files:

src/htmlAnnotator.unit.test.ts — tests for:

- Simple element at offset returns correct elementId and injects data-hover-id
- Offset outside any element returns null
- Nested elements: returns deepest match
- Priority: id attr > data-testid > data-component > structural path
- Structural path uses :nth-of-type() for same-tag siblings
- Quote-respecting > scanner in attribute injection

src/fiberScoring.unit.test.ts — tests for:

- Exact match = 0, before target = small penalty, after target = 3x penalty
- Monotonic degradation with distance
- Before-cursor is always better than equal-distance after-cursor

src/cssInliner.unit.test.ts — tests with fixture files in src/**fixtures**/:

- HTML with no links: unchanged
- Local stylesheet link: inlined as <style>
- External URL: left untouched
- Missing file: graceful fallback (no crash)
- One level of @import resolution

Files: modify src/htmlAnnotator.ts (add exports), create src/htmlAnnotator.unit.test.ts, create
src/fiberScoring.unit.test.ts, create src/cssInliner.unit.test.ts, create src/**fixtures**/ with test CSS/HTML
files

Step 6: Fix sync I/O in hover path

- src/hoverProvider.ts: make buildHover async, replace fs.readFileSync (line 146) with await
  fs.promises.readFile. Add await at all call sites.
- src/imageStore.ts: make createImageStore async (async initial load, async set()). Keep get() synchronous — it
  reads from the in-memory cache and uses existsSync for stale-entry validation, which is acceptable.
- src/extension.ts: await createImageStore().

Files: modify src/hoverProvider.ts, modify src/imageStore.ts, modify src/extension.ts

Step 7: Consolidate logging

Overhaul src/logger.ts:

- Add initLogger() function (called from activate()) that creates a VS Code Output channel and clears the debug
  log file. Moves the current module-level side effect into an explicit init.
- Export debug, info, warn, error functions. debug writes to file only; info/warn/error write to both the Output
  channel and the file.
- Keep log as an alias for debug (backward compat with devServerRenderer).

Replace bare console.log/console.error/console.warn across all source files with logger calls:

- hoverProvider.ts: console.log → info, console.error → error
- cssInliner.ts: console.warn → warn
- extension.ts: console.error → error

Files: modify src/logger.ts, modify src/extension.ts, modify src/hoverProvider.ts, modify src/cssInliner.ts,
modify src/devServerRenderer.ts

Step 8: Remove dead code, fix detector caching

- Delete src/imageServer.ts — not imported anywhere, serves .png but extension produces .jpeg. Update CLAUDE.md
  architecture table.
- Fix src/devServerDetector.ts — add 30-second cache TTL. Before returning a cached URL, verify it's still alive.
  If the server stopped, clear cache and re-probe. Export clearDetectorCache() for testing.

Files: delete src/imageServer.ts, modify src/devServerDetector.ts, modify CLAUDE.md

Step 9: Convert inspect-fibers to TypeScript

Add tsx as a dev dependency. Rename scripts/inspect-fibers.js to scripts/inspect-fibers.ts. Import shared modules
(JSX_DEV_RUNTIME_PATCH from jsxDevPatch, constants from screenshotConstants, scoreFiber from fiberScoring). The
page.evaluate callbacks still have inline copies of fiber helpers (browser boundary), but Node-side duplication
is eliminated.

Add "inspect-fibers": "tsx scripts/inspect-fibers.ts" script to package.json. Update the SKILL.md to use the new
command.

Files: rename+rewrite scripts/inspect-fibers.ts, modify package.json, modify
.claude/skills/inspect-fibers/SKILL.md

Commit Sequence

Each commit passes pnpm run compile:

1.  refactor(screenshot): extract shared constants into screenshotConstants.ts (Step 1)
2.  refactor(screenshot): extract adaptive quality pipeline into screenshotPipeline.ts (Step 2)
3.  refactor(fiber): extract scoreFiber and JSX patch into shared modules (Step 3)
4.  test(infra): add vitest for unit testing pure functions (Step 4)
5.  test(unit): add tests for htmlAnnotator, fiberScoring, cssInliner (Step 5)
6.  fix(perf): convert sync I/O to async in hoverProvider and imageStore (Step 6)
7.  refactor(logger): consolidate logging with levels and Output channel (Step 7)
8.  chore: remove dead imageServer.ts, fix devServerDetector cache TTL (Step 8)
9.  chore(scripts): convert inspect-fibers to TypeScript, consume shared modules (Step 9)

- Manual test: F5 with React dev server running → hover over JSX → preview appears
